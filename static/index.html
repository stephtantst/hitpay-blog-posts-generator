How this tool works
What it does

HitPay Blog Generator is an internal content tool for the HitPay marketing team. It uses Claude AI to write SEO-optimised, market-accurate blog posts for Southeast Asia — then lets you manage the full editorial workflow from draft to publication.

Posts are written for three markets: 🇸🇬 SG 🇲🇾 MY 🇵🇭 PH — or all three at once if no market is selected. Finished posts export as CSV for direct import into Framer CMS.

Data sources used during generation
1
HitPay MCP server — queries the internal knowledge base across three categories: general product guide. Returns up to 11 results per generation run for factual grounding on HitPay features, flows, and merchant use cases.
2
Product documentation — hitpay_docs.md is split into sections and scored by keyword relevance. The top matching sections (up to 30,000 chars) are injected so the AI can reference accurate feature names, flows, and pricing.
3
Competitor research database — competitors/*.json stores profiled payment competitors (features, markets, weaknesses). Relevant profiles are matched by keyword and market, then given to the AI as positioning context.
4
Internal blog link index — blog_links.yaml lists all published HitPay blog post URLs with topic tags. The AI picks exactly 5 relevant links and weaves them as natural in-content hyperlinks — never as a list.
What the AI is instructed to do

Every post is generated with a detailed system prompt. Key rules baked in:

  • Lead with the reader's problem — not HitPay's features. The product gets mentioned in 1–2 sections at most.
  • 900–1,200 words with 3–5 H2 sections and sparingly-used H3s. No H1 (added by CMS).
  • No marketing language — words like "seamlessly", "revolutionise", "game-changer", and "cutting-edge" are explicitly banned.
  • Market-accurate payment methods — the AI receives a per-market list of valid local and cross-border methods, plus a fact-check list of methods it must not attribute to the wrong market (e.g. PayNow is SG-only).
  • Specific local references — district and city names are required per market (Tanjong Pagar, Bangsar, BGC).
  • No fabricated stats or testimonials — all claims must come from the knowledge base or system prompt.
  • Exactly 5 internal backlinks woven naturally into the content using descriptive anchor text.
  • Payout messaging — always "next business day in SG, MY & PH", never just "T+1".
  • Output format — the AI returns a single JSON object with title slug meta_title meta_description overview categories tags content.
Generation pipeline (in order)
1
Query HitPay MCP knowledge base (general + product + guide)
2
Score and load relevant product doc sections from hitpay_docs.md
3
Match and load competitor profiles by keyword + market
4
Build internal blog link reference from blog_links.yaml
5
Send all context + keyword + market focus to claude-opus with the full system prompt
6
Parse JSON response → save to database + markdown file → open in editor
Workflow statuses
  • ✏️ Writing — draft in progress
  • 🔵 Ready to Publish — reviewed and approved
  • ✅ Published — live on the blog

Posts can be moved between statuses via the status dropdown in the editor.
